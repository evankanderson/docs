Knative for Developers

Knative gets Kubernetes out of your way.

A simple on-ramp to Kubernetes:

  • Knative simplifies Kubernetes without hiding it. You can dive deeper when you're ready.
  • Portable. Available from many vendors, and installable on any Kubernetes platform.

Knative Serving is Kubernetes for 12-factor apps:

  • Easily convert a Kubernetes Deployment to Knative; it's mostly deleting YAML!
  • By focusing on 12-factor apps, we can reduce configuration and add features:
  • Automatically scale instances based on incoming requests, including scale to zero.
  • Automatic provisioning of DNS and TLS. No need to file a ticket to set up a new service.
  • Manage request concurrency explicitly. Scale single-threaded or multi-threaded code seamlessly.
  • Support for advanced deployment strategies: canarying, incremental rollout, and fast rollback.

Serving documentation

Knative Eventing is an event-handling toolbox for Kubernetes:

  • Declarative event routing and management. Manage event routing as configuration, not in your application's code.
  • CloudEvents standard. Knative Eventing builds on the CloudEvents standard to manage events consistently.
  • Decouple producers and consumers. Knative handles asynchronous event routing, so your code can focus on business logic.
  • Dozens of off-the-shelf event sources and integrations. Integrate with AWS, GCP, Azure and more without writing code.
  • Easy to add new integrations. Sending events to Knative is as simple as POSTing a CloudEvent.
